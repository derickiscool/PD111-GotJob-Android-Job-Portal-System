# 5.3 Security Requirements

## 5.3.1 Description
This section specifies the non-functional security constraints that the system must adhere to. This includes protecting sensitive user data from unauthorized access, enforcing strict authentication protocols, and ensuring compliance with regulatory data protection laws (such as PDPA data retention and anonymization policies).

## 5.3.2 Security Constraints
The following requirements dictate the security, authentication, and compliance behaviors of the system:

* **[SRS18]** The system shall store uploaded CV and resume files securely in the database to prevent unauthorised access. (Essential)
* **[SRS19]** The system shall enforce secure authentication during registration and logging in. (Essential)

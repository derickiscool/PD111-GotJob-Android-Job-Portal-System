# 4.3 Software Interfaces

## 4.3.1 Description
This section defines the technical constraints for how the system interacts with backend services, databases, and middleware. It dictates the architectural boundaries and data storage technologies the development team must adhere to.

## 4.3.2 Interface Requirements
The following requirements govern the API, database, and authentication services:

* **[SRS15]** The system shall store users, recruiters, jobs, applications, chat messages, and notifications in MSSQL.  (Essential)
* **[SRS16]** The system shall provide an authentication service supporting login and registration for job seekers and recruiters. (Essential)
* **[SRS17]** The system shall access MSSQL only through the middleware/API layer.  (Essential)
